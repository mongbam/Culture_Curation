# 공공 오픈 API 연동형 전시 스케줄러 MVP (Culture Curation)

HTML, CSS, Vanilla JavaScript 기반으로 동작하는 가볍고 직관적인 **Static Web 전시 스케줄러 MVP**입니다. 
전국의 국공립 미술관 및 문화시설의 전시 정보, 무료/할인 혜택, 문화가 있는 날 이벤트 정보를 조회하고 개인용 주말 캘린더를 꾸릴 수 있는 서비스입니다.

---

## 1. 프로젝트 핵심 기능

- **자체 제작 SVG 격자 지도**: 외부 무거운 지도 SDK 대신 17개 전국 광역 지방자치단체 격자형 지도를 통해 지역별 전시 수 및 무료 전시 개수를 실시간으로 시각화합니다.
- **실시간 API 연동 및 Graceful Fallback**: 공공데이터포털 API Key를 입력하여 실시간 데이터를 조회할 수 있으며, 키가 없거나 네트워크 오류, CORS 등의 예외 상황이 발생하면 준비된 `data/fallback-exhibitions.json` 예시 데이터로 즉각 대체되어 서비스 중단 없이 작동합니다.
- **localStorage 기반 캘린더 저장**: 마음에 드는 전시를 저장해 나만의 주말 방문 후보를 구성하고 예상 총비용 및 무료 전시 비중 등의 통계를 계산합니다. (서버/DB 연동 불필요)
- **반응형 웹 지원 (모바일 우선)**: 360px 너비의 모바일 화면부터 데스크톱 2단 레이아웃까지 깨짐 없는 인터페이스를 제공하며, 버튼 터치 반경 44px 이상을 유지하여 터치 편의성을 극대화했습니다.
- **웹 접근성 표준 준수**: 명도 대비 확보, 스크린 리더 대응(`aria-label`), SVG 지역 키보드 초점 포커스(`tabindex="0"`, `role="button"`), `ESC` 키 모달 닫기 등을 지원합니다.

## 3. 실행 방법

본 프로젝트는 순수 Static Web 기술로만 작성되었습니다. 별도의 빌드 과정(npm install, webpack, vite 등)이나 Node.js 서버 실행이 필요하지 않습니다.

### 방법 1: 단순 더블 클릭
1. 프로젝트 루트의 `index.html` 파일을 크롬(Chrome)이나 사파리(Safari) 등 웹 브라우저로 직접 더블 클릭하여 실행합니다.

### 방법 2: 로컬 웹 서버 사용 (CORS 및 Fetch API 호환 권장)
브라우저의 보안 정책상 파일 프로토콜(`file:///`) 환경에서 로컬 JSON 파일 fetch가 차단되는 경우, 로컬 개발용 웹 서버를 통해 실행하는 것을 권장합니다.
- **VS Code Live Server**: VS Code 사용 시 `Live Server` 확장을 설치한 후 우측 하단의 `Go Live`를 클릭하여 기동합니다.
- **Python 임시 서버**: 터미널에서 프로젝트 폴더로 이동한 후 아래 명령어를 입력합니다.
  ```bash
  python -m http.server 8000
  ```
  이후 브라우저에서 `http://localhost:8000`에 접속합니다.

## 4. 공공 API Key 설정 및 호출 방법

본 MVP는 공공데이터포털의 **한국문화정보원 문화포털 공공데이터 API** 및 **한국관광공사 국문 관광정보 서비스(축제/행사)** 연동 규격을 제공합니다.

1. [공공데이터포털(data.go.kr)](https://www.data.go.kr)에 로그인하여 원하는 문화/행사 API를 신청하고 **인증키(Encoding/Decoding Service Key)**를 발급받습니다.
2. 실행된 웹 브라우저 화면 우측 상단의 `공공데이터포털 API Key 입력` 란에 발급받은 API 키를 붙여넣은 뒤 **적용** 버튼을 누릅니다.
3. 인증키가 유효한 경우 실시간으로 데이터 fetch를 실행하며, 연동 상태 배지가 **🟢 실시간 공공 API 데이터 연동**으로 전환됩니다.
4. 상용 서비스 배포 시에는 API Key 노출 방지를 위해 반드시 서버 측 프록시(Proxy) 또는 Serverless Function을 구성하여 API Key를 숨겨 호출하여야 합니다.

## 5. 데이터 출처 및 저작권 정책

- **데이터 제공처**: 공공데이터포털(data.go.kr) 공공누리(KOGL) 제1유형 조건 데이터를 기준으로 합니다.
- **포스터 이미지 배제 배경**: 각 박물관/미술관별 포스터 이미지의 저작권 및 2차 배포 권한이 불명확하고 무단 크롤링 방지 정책을 고려하여, 본 MVP에서는 어떠한 외부 전시 이미지도 사용하지 않습니다. 
- **대체 카드 디자인**: 이미지 대신 직관적인 **카테고리 텍스트, 색상 배지, CSS 패턴, 메타데이터 아이콘**을 통하여 정제된 정보를 세련되게 시각화합니다.
- **출처 표기 의무화**: 모든 전시 정보 카드의 하단 영역에 데이터 출처 제공처 명칭(예: 국립현대미술관) 및 최종 확인/업데이트 일자, 원문 링크를 직접 표기해 신뢰성을 부여하고 투명성을 유지합니다.

## 6. MVP 한계 및 향후 개선 방향

### MVP의 제약 조건
- **API CORS 한계**: 브라우저에서 직접 공공 데이터를 호출하므로, 공공 API 서버 측에서 브라우저 직접 호출에 대한 CORS 허용 정책을 갖추지 않은 경우 보안 에러로 인해 Fallback 데이터로 전환됩니다.
- **정확한 비용 계산 불가**: 공공 API의 비용 텍스트 품질이 표준화되어 있지 않아(예: "성인 5,000원 / 청소년 3,000원", "무료(단, 기획전 별도)") 유료 전시의 경우 단순 숫자 추출 방식을 사용하며, 할인의 경우 "현장 확인 필요"로 안내하고 총합 계산에서 제외합니다.
- **위치 좌표**: SVG 격자 지도는 위도/경도가 아닌 광역 지자체 격자 셀 기준 필터링을 지원합니다.

### 향후 개선 방향
- **백엔드 API 프록시 서버 구축**: CORS 차단 문제를 방지하고 API Key를 보호하기 위한 Node.js / Vercel Serverless 프록시 API를 도입합니다.
- **자연어 기반 요금 파서 고도화**: LLM 또는 정규식 패턴 그룹을 통해 성인/소인/단체별 복잡한 입장료 형식을 완벽하게 정수형으로 추출해내는 로직을 도입합니다.
- **카카오/네이버 맵 API 연동**: 좌표(`lat`, `lng`)가 확보되는 공공 전시에 대해 마커를 표시하는 기능 및 길 찾기 연동을 개선합니다.
